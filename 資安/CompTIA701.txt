Which of the following describes two-factor authentication?
翻譯：以下哪一個描述了 雙因素驗證（2FA, Two-Factor Authentication）？

原文：The Answer: D. A Windows Domain requires a password and smart card
翻譯：答案：D. Windows 網域需要 密碼 + 智慧卡
👉 這裡的雙因素是：
密碼 = 你知道的東西 (Something you know)
智慧卡 = 你擁有的東西 (Something you have)

A. A printer uses a password and a PIN
翻譯：A. 印表機使用密碼與 PIN 碼
👉 密碼和 PIN 都屬於 你知道的東西，所以只是一種因素（單因素），不是雙因素。
👉 指紋是 你是什麼 (Something you are)，屬於單一生物特徵驗證，不是雙因素。

C. An application requires a pseudo(偽)-random code
翻譯：C. 一個應用程式需要輸入隨機碼
👉 隨機碼通常來自 硬體金鑰或手機 App，屬於 你擁有的東西 (Something you have)，也是單因素。

✅ 雙因素驗證 (2FA) = 必須來自 不同類別 的兩種因素。

During a morning login process, a user's laptop was moved to a private VLAN and a series of updates were automatically installed.
翻譯：在早晨的登入過程中，一名使用者的筆電被移到一個 私有 VLAN，並自動安裝了一系列更新。
Private VLAN：一種網路隔離機制，把設備臨時放到受控區域，避免在不安全狀態下接觸主網路。
這通常屬於 Network Access Control (NAC) 的一部分，用來驗證裝置是否符合安全規範（patch、AV、加密）。
原文：Which of the following would describe this process?
翻譯：以下哪一個最能描述這個過程？

翻譯：答案：B. 設定強制執行（Configuration enforcement）
翻譯：許多組織會在登入過程中執行 姿態評估（Posture assessment），以確認裝置是否具備正確的安全控制措施。如果裝置未通過評估，系統會將其隔離，並安裝缺少的安全更新。
Posture Assessment：檢查裝置的安全狀態（是否有最新更新、防毒軟體、防火牆等）。
Configuration Enforcement：自動確保裝置符合安全設定要求，例如更新補丁、安裝必要軟體。

翻譯：C. 除役（Decommissioning）
除役是指將設備永久移除出網路。但題目中的筆電在完成更新後仍可回到網路，因此不是除役。

翻譯：某家公司將一個 SCADA 系統 放置在一個分段的網路中，並限制來自其他公司內部網路的存取。
SCADA (Supervisory Control and Data Acquisition，監控與資料擷取系統)：主要用於工業控制系統（ICS），監控與控制工廠、能源、交通、水處理等關鍵基礎設施。
Network Segmentation（網路分段）：把網路分割成不同區域，降低攻擊面並限制橫向移動。
原文：Which of the following would describe this process?
翻譯：以下哪一個最能描述這個做法？

原文：The Answer: D. Hardening
翻譯：答案：D. 系統強化（Hardening）
翻譯：針對工業用 SCADA 系統 的強化過程，可能會包括 網路分段、額外的 防火牆控制，以及 存取控制清單（ACL） 的實作。

原文：C. Data retention
翻譯：C. 資料保存（Data retention）
資料保存是指長期儲存重要資訊的策略與政策。但題目中沒有涉及強制保存資料的情境，所以不符合。

翻譯：系統管理員被叫去處理一台遭受惡意軟體感染的系統。

翻譯：作為事件回應流程的一部分，管理員已將作業系統還原成一個已知安全的版本。
Imaging（系統映像/還原）：用一個乾淨的、可信任的映像檔（image）覆蓋現有系統，以移除感染。
原文：Which of these incident response steps is the administrator following?
翻譯：管理員正在執行事件回應流程中的哪一個步驟？

翻譯：答案：B. 復原（Recovery）

原文：D. Containment
翻譯：D. 隔離（Containment）
隔離階段會將受感染的系統與其他設備隔離，以防止惡意軟體進一步擴散。隔離通常在偵測到感染後立即進行。

A company has decided to perform a disaster recovery exercise during an annual meeting with the IT directors and senior directors.
翻譯：一間公司決定在年度會議中，與 IT 部門主管及高階主管一起進行災難復原演練。
原文：A simulated disaster will be presented, and the participants(參與者) will discuss the logistics and processes required to resolve the disaster.
翻譯：會議中將會模擬一場災難，並由參與者討論處理該災難所需的作業細節與流程。
Logistics（作業規劃/後勤流程）：確保資源、程序、人員能協調運作的細節安排。
原文：Which of the following would BEST describe this exercise?
翻譯：下列哪一項最能描述這種演練？

翻譯：答案：D. 桌上演練（Tabletop exercise）
翻譯：桌上演練讓災難復原小組能夠在不進行全面性演練的情況下，評估並規劃災難復原流程。
Tabletop exercise：模擬災難場景，僅透過討論與情境分析，而不實際操作系統或中斷業務。

原文：A. Capacity planning
翻譯：A. 容量規劃（Capacity planning）
容量規劃是用來決定完成特定任務需要多少資源。正式的桌上演練通常不會包含容量規劃分析。

翻譯：B. 業務影響分析（Business impact analysis, BIA）
業務影響分析通常在災難復原計劃的制定過程中進行。一旦災難已經發生，就很難再完成準確的影響分析。

翻譯：C. 業務連續性（Continuity of operations, COOP）
當發生中斷時，通常會有備援計劃以確保業務持續。這種計劃適用於各類重大中斷事件，而非專門針對災難復原測試。

：An organization has identified a significant vulnerability in an Internet-facing firewall.
翻譯：一個組織發現其面向互聯網的防火牆存在重大漏洞。
Internet-facing firewall：直接面向外部網路的防火牆，通常保護內部網路不受外部攻擊。
漏洞（Vulnerability）：系統或設備中可能被利用的安全缺陷。

原文：The firewall company has stated(指出，表示) the firewall is no longer available for sale and there are no plans to create a patch for this vulnerability.
翻譯：防火牆公司表示該防火牆已停止銷售，且無計畫針對此漏洞發布補丁。
補充：這表示該設備已停止支援，未來也無法透過官方更新修復漏洞。

原文：Which of the following would BEST describe this issue?
翻譯：下列哪一個最能描述這個問題？

翻譯：答案：A. 產品生命週期結束（End-of-life, EOL）

End-of-life (EOL)：產品停止銷售、停止維護或更新，可能造成安全風險，建議更換或升級。

翻譯：D. 作業系統不相容（Incompatible OS）

原文：A security administrator is building new servers and security systems to get all of the financial systems back online.
翻譯：資安管理員正在建立新的伺服器與安全系統，以將所有財務系統恢復上線。
恢復系統（Recovery of systems）：在事件後重新部署硬體與軟體，使業務正常運作。

原文：Which part of the incident response process would BEST describe these actions?
翻譯：事件處理流程中，哪個階段最能描述這些行動？

翻譯：答案：C. 復原（Recovery）
翻譯：資料外洩事件後的復原可能採取分階段方式，可能需要數個月才能完成。
分階段復原（Phased recovery）：先恢復最關鍵的系統，再逐步恢復次要系統，以降低業務中斷風險。

翻譯：A. 經驗回顧（Lessons learned）

翻譯：B. 隔離（Containment）

翻譯：D. 分析（Analysis）
分析階段可以包含日誌檔與警示的分析，這些資料來源可以幫助偵測潛在攻擊或提供攻擊正在進行的證據。

A technician is applying a series of patches to fifty web servers during a scheduled maintenance window.
翻譯：一名技術人員正在排定的維護時段內，對五十台 Web 伺服器套用一系列補丁。
補丁（Patch）：用於修復軟體漏洞或提升功能的小型程式更新。
維護時段（Maintenance window）：事先排定的系統維護時間，以避免影響使用者。
原文：After patching and rebooting the first server, the web service fails with a critical error.
翻譯：在對第一台伺服器套用補丁並重新啟動後，Web 服務發生重大錯誤而無法運作。
重大錯誤（Critical error）：會導致服務無法正常運作，通常需立即處理。
原文：Which of the following should the technician do NEXT?
翻譯：技術人員接下來應該做什麼？

翻譯：答案：B. 依照回退計畫（Backout plan）中的步驟進行
回退計畫（Backout plan）：在變更控制流程中，如果發現無法修復的錯誤，可將系統回復到變更前的設定或狀態的計畫。
翻譯：變更控制流程中的回退計畫提供了如何在變更過程中發現無法修復的錯誤時，回復至先前設定的資訊。

原文：A. Contact the stakeholders regarding the outage
翻譯：A. 通知利害關係人有關停機事件
利害關係人通常不需要在維護時段中收到每個步驟的詳細通知。變更的最終結果可以在維護完成後再告知利害關係人。

原文：C. Test the upgrade process in the lab
翻譯：C. 在實驗室測試升級流程
變更控制流程中的測試階段應在維護時段開始前完成。一旦維護時段開始，就太晚無法在實驗室進行額外測試。

原文：D. Evaluate the impact analysis associated with the change
翻譯：D. 評估與變更相關的影響分析
影響分析用來評估提出變更的風險。這份分析通常在變更控制核准前完成，在維護時段中用來排錯並不太有用。

Which of the following is used to describe how cautious(謹慎) an organization might be to taking a specific risk?
翻譯：下列哪一個用來描述組織對承擔特定風險的謹慎程度？
補充：題目是在問「組織面對風險時的態度或容忍度」。

翻譯：答案：A. 風險偏好（Risk appetite）
翻譯：風險偏好是對可接受風險程度的一個廣泛描述。組織的風險偏好態度可能偏保守，也可能較積極，願意承擔額外風險。
風險偏好（Risk appetite）：衡量組織願意接受的風險程度。
保守（conservative）：傾向避免風險。
積極（expansionary / aggressive）：願意承擔更多風險以追求機會。

翻譯：B. 風險登錄（Risk register）
風險登錄用於識別並記錄專案計畫中每個步驟相關的風險，但它不是用來描述組織對每個風險的謹慎程度。
補充：風險登錄通常包含風險名稱、影響程度、可能性、責任人等。

Which of the following considerations are MOST commonly associated with a hybrid cloud model?
翻譯：下列哪一項考量最常與混合雲（hybrid cloud）模型相關？
混合雲（Hybrid Cloud）：同時使用私有雲（private cloud）與公有雲（public cloud）的架構。

翻譯：答案：C. 網路防護不一致（Network protection mismatches）
翻譯：混合雲包含多個私有雲或公有雲，這會增加整體基礎設施的複雜性，而且很容易在多個雲服務提供者之間不小心使用不同的認證選項或使用者權限設定。
補充：
認證選項（authentication options）：指使用者登入或驗證方式，如 OAuth、SAML、密碼等。
使用者權限（user permissions）：控制誰能存取哪個資源。
網路防護不一致：在多雲環境中，不同雲提供者可能使用不同的防火牆規則、存取控制或安全政策，容易造成漏洞。

翻譯：A. 微服務中斷（Microservice outages）
微服務用於建立可擴展且具韌性的應用程式實例，但微服務的可用性並非混合雲模型特有的問題。

翻譯：B. 物聯網（IoT）支援
物聯網支援在許多雲端基礎設施模型中都可用，並非混合雲特有。

翻譯：D. 容器化備份（Containerization backups）
容器化提供高效的應用程式部署方式，但容器的使用與備份並非混合雲特有。
容器化（Containerization）：使用容器（如 Docker）封裝應用程式及其依賴，以便跨環境一致部署。

C. 驗證
驗證指的是確認個人或系統身份的過程。帳號與密碼是常見的驗證方法，但數位簽章並不常用於身份驗證。

可用性是指授權使用者能夠存取資料的能力。數位簽章並不會提供與資料可用性相關的功能。

也就是說，單純抓封包只是「偷看加密流量」，憑證仍然有效。

修改封包 / 插入自己的伺服器
你必須用自己的憑證去回應使用者

A company's email server has received an email from a third-party,
→ 一家公司的電子郵件伺服器收到了來自第三方的電子郵件，

→ 但是發信的來源伺服器與授權設備清單不符。
Which of the following would determine the disposition(處理方式) of this message?
→ 下列哪一項可以決定這封郵件的處理方式（是否允許、阻擋或標記為垃圾郵件）？
❍A. SPF❍B. NAC❍C. DMARC❍D. DKIM

答案：C. DMARC
DMARC（基於網域的郵件驗證、回報與一致性規範）可以指定垃圾郵件的處理方式。
合法的來源郵件網域擁有者可以決定這些郵件是被 接受、放入 垃圾郵件夾，或 拒收。
DMARC：一種電子郵件驗證技術，用來防止 假冒發信（Email Spoofing），同時告訴接收伺服器應該如何處理可疑郵件。

A. SPF
SPF（寄件者政策框架）是一個列出某個網域所有授權郵件伺服器的清單。
所有合法郵件都應該從 SPF 配置中列出的伺服器之一發送。
SPF 主要用來 驗證郵件發信伺服器是否合法，但不決定垃圾郵件的最終處理方式。

B. NAC
NAC（網路存取控制）是一種限制網路存取權限的方法，只允許授權使用者接入網路。
NAC 通常不會用來管理電子郵件的傳送。
NAC 屬於網路層的存取控制技術，與郵件安全無直接關係。

D. DKIM
DKIM（網域簽名郵件）提供一種方式，用來驗證來自特定郵件伺服器的所有數位簽章郵件。
DKIM 並不決定接收伺服器如何分類這些已簽章的郵件。
DKIM 用來保證郵件內容未被篡改和確認發信者身份，但不決定垃圾郵件或阻擋策略。

不直接存取或攻擊目標系統，只利用公開資訊蒐集情報，像是 OSINT（Open Source Intelligence，開放來源情報）。

A. 漏洞掃描（Vulnerability scanning）
一些主動偵察測試會直接詢問系統，以確認漏洞是否存在。
需要對目標系統直接互動，通常會觸發系統回應。

C. 供應鏈分析（Supply chain analysis）
供應鏈分析會檢視與供應商相關的安全性，而這種分析不會提供有關公司自身伺服器與資料的資訊。
重點在評估合作夥伴或供應商的安全性，而非目標公司內部系統。

D. 法規稽核（Regulatory audit）
法規稽核是基於現行法律或私人準則進行的詳細安全分析。
法規稽核通常需要存取內部系統與資料。
通常是合規檢查（compliance-遵守 check），需要內部資料，因此不是被動偵察。

During the login process, your phone receives a text message with a one-time passcode => Something you have
You enter your PIN to make a deposit(訂金) into an ATM => Something you know
You can use your fingerprint to unlock the door to the data center => Something you are
Your login will not work unless you are connected to the VPN => Somewhere you are

1. Something you have（你擁有的東西）
定義：使用者必須持有的物理或電子物件來驗證身份。
範例：智慧卡（smart card）、OTP 令牌（token）、手機收到的驗證碼、USB 安全金鑰（YubiKey）。

2. Something you know（你知道的東西）
定義：使用者必須知道的秘密資訊來驗證身份。
範例：密碼（password）、PIN、答案問題（security question）。

3. Something you are（你是誰）
定義：基於使用者的生物特徵來驗證身份。
範例：指紋、虹膜掃描、臉部辨識、聲紋、生物特徵認證。

4. Somewhere you are（你在哪裡）
定義：根據使用者的地理位置或裝置位置來驗證身份。
範例：GPS 定位、IP 位址檢查、Wi-Fi 網路位置、VPN 登入限制。

💡 補充：
多因素認證（MFA, Multi-Factor Authentication）就是要求 至少兩種不同類型的因素，
例如「Something you have + Something you know」或「Something you know + Something you are」，大幅提升安全性。

A guard checks the identification of all visitors / 檢查每個訪客的身分證 => Operational
All returns must be approved by a Vice President(副總) => Managerial
A generator is used during a power outage(停電) => Physical
Building doors can be unlocked with an access card => Physical
System logs are transferred automatically to a SIEM => Technical

Operational：人員每天執行的程序
Managerial：政策、授權、策略決策
Physical：保護建築、設備、人員
Technical：依靠技術或系統自動執行的控制

✅ 總結：
Control categories describe the type of security applied to a task or event.
控制類別描述了應用於某項任務或事件的安全類型。

Operational controls are often implemented by people instead of systems.
操作性控制（Operational controls）通常由人員執行，而非系統自動化完成。

Security guards and awareness programs are examples of an operational control.
保全人員和資安意識訓練計畫（awareness programs）都是操作性控制的範例。

Managerial controls are administrative controls associated with security design and implementation.
管理性控制（Managerial controls）是與安全設計與實施相關的管理性控制（administrative controls）。

A set of policies and procedures would be an example of a managerial control.
一套政策與作業程序（policies and procedures）就是管理性控制的範例。

Physical controls are used to limit physical access.
實體控制（Physical controls）用於限制物理存取。

Badge readers, fences, and guard shacks are categorized as physical controls.
門禁讀卡器（badge readers）、圍欄（fences）以及保全崗亭（guard shacks）都屬於實體控制。

Technical controls are implemented using systems.
技術控制（Technical controls）是透過系統來實施的控制措施。

Operating system controls, firewalls, and automated processes are considered technical controls.
作業系統控制（Operating system controls）、防火牆（firewalls）以及自動化流程（automated processes）都被視為技術控制。

Attacker obtains bank account number  and birth date by calling the victim?
	=> Vishing (與音釣魚)
Attacker accesses a database directly  from a web browser
	=> Injection (資料庫注入)
Attacker intercepts all communication between a client and a web server
	=> On-path (中間人攻擊)
Multiple attackers overwhelm a web server
	=> DDoS (癱瘓攻擊)
Attacker obtains a list of all login credentials used  over the last 24 hours
	=> Keylogger (間諜攻擊)
	
On-path (MITM)：攔截並可能篡改雙方通訊。
RFID cloning：複製 RFID 晶片資料以冒用身份。
Keylogger：記錄鍵入資訊（密碼、帳號等）以竊取資料。
Vishing：透過語音電話詐騙取得敏感資訊。
Rootkit：隱藏於系統核心的惡意軟體，持久控制系統。
DDoS：用大量流量或請求癱瘓目標服務。
Injection：將惡意輸入當作程式碼或查詢執行，操控後端。
Supply chain attack：滲透供應鏈或第三方元件，藉信任渠道影響目標。
