🧮 1. 一台電腦最多可以開幾個 port？
✅ 答案：65535 個 TCP 或 UDP port
port 是 16-bit 整數，所以理論上： 0 ~ 65535 共 65536 個
其中：
0 號是保留的，不能用
1~1023 是「知名端口（well-known ports）」，需要 root 權限才能開啟（在 Linux 下）
1024~65535 是「動態 / 私人端口」，可由應用程式自由使用

🧠 2. 怎麼算出來的？
✳️ port 號是 16-bit： 2^16 = 65536
代表最多可以表示 0 到 65535 的數字 → 所以總共 65536 個 port（扣掉保留的 port 0，最多可用 65535 個）

🔄 3. 如果某個 port 被用了，別人可以再用嗎？
❌ 不可以
=> 在 127.0.0.1 上的 port 8080 已經有程式（例如 Nginx）綁定了
那麼其他程式就 不能再綁定 127.0.0.1:8080
會出現錯誤：Address already in use

✅Nmap（Network Mapper）是一款用來掃描網路設備的開源工具，主要用於：
探測目標主機是否在線（ping 掃描）
掃描目標主機開啟的端口（port scanning）
判斷服務版本（version detection）
推測目標作業系統（OS fingerprinting）

Nmap 的原理大概分幾部分：
🔄掃描目標是否存活（Ping 掃描）
透過 ICMP Echo Request 或 TCP SYN/ACK 封包確認目標是否回應
節省時間，避免掃描不存在的主機

🔄端口掃描（Port Scanning）
送出不同類型的封包到目標的各個 port，觀察回應來判斷該 port 是開啟、過濾還是關閉
常用掃描技術：
TCP SYN 掃描（半開掃描）：只發送 SYN 封包，看是否回 SYN-ACK 來判定端口開啟
TCP Connect 掃描：嘗試建立完整 TCP 連線（SYN, SYN-ACK, ACK）
UDP 掃描：透過 UDP 封包，觀察回應來判定開啟狀態

🔄服務版本探測（Version Detection）
Nmap 會送出特製的封包給開啟的 port，藉由服務回應的字串、協定細節判斷服務類型與版本
例如 HTTP Server 回應的 header 中 Server 字串，FTP 回應碼格式，等等

🔄作業系統指紋辨識（OS Fingerprinting）
Nmap 透過分析目標主機對多種不同 TCP/IP 封包的回應細節（例如 TCP 頭 flags、IP 頭選項、TTL、Window Size 等），對比其資料庫中的已知作業系統行為模式，推測作業系統種類與版本範圍
這叫做TCP/IP 堆疊指紋（TCP/IP Stack Fingerprinting）

🔄為什麼 Nmap 能抓到這麼詳細的資料？
原因有三：
自訂精密封包	Nmap 不只是「連接」，而是發送特別設計的封包（比如奇怪的 TCP Flag 組合）來測試目標回應，藉此分析更多細節
龐大且不斷更新的資料庫	Nmap 維護有龐大的指紋資料庫（OS、服務版本），透過比對回應細節做精準推斷
多種掃描技術結合	不只掃端口，也掃服務版本、作業系統，並支援腳本（NSE，Nmap Scripting Engine）進行更多自訂探測
