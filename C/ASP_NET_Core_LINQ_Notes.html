ASP.NET Core 與 LINQ 筆記整理

LINQ 與 EF Core 的安全性

LINQ 本身不是資料庫查詢，但在搭配 Entity Framework Core (EF Core) 時會安全。

  • LINQ 是查詢語法，可用在 Array / List（不會有 SQL Injection）
  • EF Core 會將 LINQ 查詢自動轉成參數化 SQL，阻擋 SQL Injection
// 使用 LINQ 查詢資料
using (var db = new AppDbContext())
{
    var user = db.Users
                 .Where(u => u.Name == "Alice")
                 .FirstOrDefault(); // 自動生成參數化 SQL
}

Entity Framework Core (EF Core)

EF Core 是 .NET 的 ORM（Object-Relational Mapper），用於將資料表映射為 C# 類別。

  • 可用物件方式操作資料庫，無需手寫 SQL
  • 支援多種資料庫：SQL Server、PostgreSQL、MySQL、SQLite…
  • DbContext：資料庫操作主體
  • DbSet<TEntity>：對應資料表

LINQ 是什麼？

LINQ（Language Integrated Query）是一種整合在 C# 語言中的查詢語法。

  • 可查詢：陣列、List、XML、JSON、資料庫
  • 兩種語法風格：方法語法（函式式）與查詢語法（類 SQL）

ASP.NET Core 與 Kestrel

ASP.NET Core 是跨平台的 Web Framework，預設使用 Kestrel 伺服器。

  • 跨平台（Windows / Linux / macOS）
  • 高效能、支援 HTTP/HTTPS
  • 可直接對外服務，或與反向代理（Nginx、IIS、Apache）搭配

[ApiController] 屬性

[ApiController] 是用來標註 Controller 為 API 控制器，不是父類別。

  • 自動模型驗證
  • 自動回傳 400 錯誤
  • 自動推斷參數綁定

REST 與 RESTful API

REST（Representational State Transfer） 是一種 Web API 設計風格。

設計原則

  • 資源導向：用名詞而非動詞
  • HTTP 方法表達動作（GET、POST、PUT、PATCH、DELETE）
  • 無狀態，每個請求獨立
  • 統一格式（JSON / XML）

HTTP 狀態碼

  • 200 OK
  • 201 Created
  • 204 No Content
  • 400 Bad Request
  • 401 Unauthorized
  • 404 Not Found
  • 500 Internal Server Error

ASP.NET 技術演進

  • ASP (Classic) → 1996，VBScript，副檔名 .asp
  • ASP.NET → .NET Framework 2.0~4.8.1，支援 Web Forms / MVC / Web API
  • ASP.NET Core → 2016 起，跨平台，整合 MVC + Web API + Razor

ASP.NET Core 在 .NET 5 起去掉「Core」字樣，改稱「ASP.NET on .NET 5+」。
